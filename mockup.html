App Store Conversion Roadmap
Framework React Native / Expo
  • Current web app is Next.js — screens would port cleanly to Expo Router (file-based routing, same mental model)
  • Tailwind → NativeWind (Tailwind classes compiled to StyleSheet — minimal rewrite)
  • Most business logic (API calls, auth, state) is reusable 1:1 — only UI primitives change
  • Mapbox has a React Native SDK (@rnmapbox/maps) — same tile styles & tokens
Native Upgrades Key Wins
  • Background GPS — expo-location with background task for continuous breadcrumb logging (currently browser-dependent)
  • Push Notifications — Expo Push / Firebase FCM replacing email for expense alerts and flag alerts
  • Camera — expo-camera / expo-image-picker for receipt capture (better UX than file input)
  • Biometric Auth — Face ID / fingerprint via expo-local-authentication
  • Offline Mode — queue clock events locally when no signal, sync on reconnect
Backend Minimal Changes
  • All existing Next.js API routes remain as-is — the mobile app just calls the same fieldmanagerpro.app/api/* endpoints
  • Auth moves from httpOnly cookie → SecureStore JWT (expo-secure-store)
  • S3 presigned URLs work identically in mobile fetch()
  • Supabase DB, Resend, Vercel Crons — no changes needed
Store Submission Timeline Est.
  • iOS (TestFlight → App Store) — requires Apple Developer account ($99/yr), Xcode build, App Store review ~1–3 days
  • Android (Play Console) — Google Play account ($25 one-time), APK/AAB build, review ~1–7 days for new apps
  • Expo EAS Build handles both iOS & Android cloud builds — no local Xcode/Android Studio required
  • Estimated dev time to MVP app: 4–6 weeks (screen porting + native features + store setup)
